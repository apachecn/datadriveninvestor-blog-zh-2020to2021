# 什么是对话式 UI？

**对话用户界面**是一种用户界面，其中主要的输入模式是口头的和非正式的(即，没有严格的结构)。值得注意的是，它不一定要完全由语音控制，它只是以一种更松散、更随意的方式进行操作，即对话。

对话式界面是优秀的老式命令行的自然延续。它们的重要进步是对话式用户界面不仅仅是做它被告知要做的事情。这是一个更舒适的工具，如果它解决了用户的问题，还会产生许多有价值的见解。

现在让我们看看对话界面是如何工作的。

# 对话界面是如何工作的？

对话式 UI 基于调用和响应方法进行操作。**自然语言处理**和**机器学习算法**是对话式 UI 设计的一部分。它们塑造了自己的投入产出特征，并在运营过程中提高了效率。

总体而言，该操作需要:

*   **自然语言处理**解释输入的文本，
*   **图像识别**对于图像和文字填充的图像，
*   **自然语言生成**的特点是提供连贯的反应。
*   **文本到语音，语音到语音**输出功能。

有三种方式可以呈现用户输入:

*   通过**文本输入**(需要自然语言处理/生成)；
*   通过**图像输入**(不太常见；通常是附加选项。需要光学字符和图像识别)；
*   通过**语音输入**(除了 NLP——需要语音识别和文本到语音转换工具)。

自然语言处理算法解释输入。下面是它的一步一步的工作方式:

*   **定义**查询的类型 aka 意图(即做某事或发现某事等。);
*   **提取**查询的重要元素(即理解对哪个对象做什么动作)；
*   **交付**通过执行请求的动作或提供响应来表示的输出。

对话界面依靠大量的用户数据来提供最有效的服务。以下方法获取数据:

*   **用户账户数据**
*   **直接输入**
*   **过去使用数据**
*   **补充数据**用于解释上下文。

这些数据的目的是:

*   操作对话平台，
*   自然语言处理平台
*   机器学习算法，
*   语音识别和生成平台。

因此，用户可以获得与他们的查询相关的结果或建议，并通过简单地说出或键入命令来简化他的工作流程。

## 什么是虚拟助理？

对话界面最常见的用法是虚拟助手。这两个术语意味着一枚硬币的两面。

*   对话式 UI 是技术；
*   虚拟助手或聊天机器人是一系列产品，它们将对话界面的潜力发挥到了极致。

“虚拟助理”(也称为“虚拟智能助理”和“人工智能助理”)的标准定义是一个**“口头操作的程序，旨在例行或应要求执行某些动作。”**

# 对话界面的类型

值得注意的是，“对话接口”是一个涵盖了几乎所有基于对话的交互服务的总称。

有些人认为对话式用户界面只是“聊天机器人”的另一个更加华丽的代名词。虽然存在直接连接，但聊天机器人仅代表一种特定类型的对话界面，它涉及用于实现其操作的对话元素，但不是由它定义的。

从用例的角度来看，有几种不同类型的对话接口:

*   **Q &一个网聊界面**——最基本的形式。它不需要自然语言处理或机器学习。它是一种算法，根据请求直接从其数据库中提供信息——通常用于导航内容和提供 FAQ 摘录。
*   **客户支持界面**(又名最常见类型)。基于调用和响应，基于模板的系统。能够提供一般信息。包括评估机器人满足传入请求的能力的另一层交互。如果这是不可能的，系统会转向人工操作员。
*   **用户参与界面** —这种类型在公司中很快变得普遍。这个机器人本质上是帮助用户浏览网站、回答基本问题和交换信息。可用于内容或产品建议。作为其功能的扩展，它还可以提供初始销售线索挖掘活动。
*   组织者界面 —这更像是一种 PDA 组织者类型。这种界面设计旨在让用户检查自己的时间表，管理待办事项，提醒不同的事情，并执行基本的操作，而无需在窗口/应用程序中跳跃。它是一个与其他服务集成的反向 bot。
*   **多功能智能虚拟助理又名人工智能助理**——这些是大型科技物联网解决方案，如亚马逊 Alexa、苹果 Siri、微软 Cortana。

# 对话界面用例

## 基本客户支持

在客户支持操作的情况下，对话界面根据请求传递信息。它可以直接取自源材料，也可以通过自然语言处理功能重新解释，以适应对话的上下文。

在更复杂的情况下，客户支持助理还可以处理通知、发票、报告和后续信息。

除此之外，系统还可以将人工操作员重定向到机器人无法触及的查询。

## 对话导航/服务指南

简化用户体验是改善客户体验的重要因素。自然语言用户界面是它实现的方式之一。

以下是您可以使用的帮助类型:

*   指导结帐过程(例如，货币交易)；
*   填写网络表单、订阅、注册等。
*   建议操作选项(下载、注册等。);
*   建议内容

助理的主要目的是收集正确的数据，并将其用于改善客户体验。

CRM 助手就是一个很好的例子。根据配置，他们可以:

*   回答问题，
*   建议选项(取决于具体情况)
*   执行特定的操作(对联系人进行分类、制作报告等。).

## 线索挖掘

从简单的客户支持开始，下一步就是挖掘潜在客户。对话式界面通过方便的对话式用户体验，尽可能多地提取有价值的信息，而不是根据请求进行操作。

原因很简单——对话是吸引用户并将他转化为客户的绝佳方式。

问题的性质可能有所不同，但核心通常是联系信息和关于用户的必要业务细节:

*   姓名，
*   位置，
*   公司的名称，
*   联系信息(电子邮件或电话等)。

然后，这些信息直接进入客户关系管理平台，用于培育销售线索，并将其转化为合法的商业机会。

## 生产力

生产力是对话式用户界面目前使用的主要领域之一。难怪——有太多的日常事务要处理。

这样的助手是连接到计算机的所有应用程序的命令行，可以理解不太严格的命令。

生产力对话界面的主要目标是简化工作过程，使其不那么混乱，并完全避免例行公事的可疑点。

例如，生产力助理可以处理基本的任务管理职责，例如:

*   任务管理——创建、分配和状态更新；
*   检索报告，促进交流；
*   时间管理——保持时间表不变，预订房间，预约，设置提醒。
*   研究—以经过处理的形式提供搜索结果，最常见的形式是摘要或摘要。

此外，这样的接口可以提供基于任务管理框架的关于性能的度量。

## 内容建议

这些天有太多的内容需要阅读。那是一个冷酷的事实。为了从碎片中找到最有价值的内容，用户需要一些额外的工具来对内容进行分类，并且只提供重要的内容。

内容推荐是对话界面的核心用例之一。通过机器学习，机器人可以根据用户的偏好和/或表达的行为来调整内容选择。

结果可以以对话的方式呈现(例如大声朗读标题)，或者以更正式的方式突出或总结内容。例如,《纽约时报》提供了以对话格式显示文章的机器人。

## 对话营销

聊天机器人在合适的营销团队手中可以成为大规模参与的武器，这不是什么大秘密。

正如电子邮件营销为品牌展示提供了一个案例，聊天机器人也可以在多个平台上做同样的事情。

这种方法不仅限于你的网站，它也适用于社交网络。这种界面的功能可能会有所不同。一般来说，是:

*   基本信息交换
*   内容交付/内容建议
*   新闻摘要
*   后续行动

这种对话式用户界面的最大好处是保持跨多个平台的存在，并通过一种不太正式的方式促进客户参与。

# 对话界面挑战

## 定义相关用例

对话接口的实现围绕着一件事——它的使用目的。

正如之前在用例部分提到的，用例可能从基本的问答到实际操作的组织者，再到强大的线索生成和营销工具。

在开始开发对话界面之前，理解你想用它做什么是很重要的。

不同类型的界面需要不同的功能，不能通过轻轻一按手腕来调整以做其他事情。

怎么做，对吧？洞察力不是凭空而来的。用例的定义来自用户自己。

成功的关键在于学习:

*   什么样的行为对用户有益？
*   某些特征如何有助于提高参与度？
*   什么级别的对话式 UI 可访问性适合目标受众？

你需要做的是:

*   **事件监控** —用户通常在你的平台上做什么样的动作。有哪些弱点？下降发生在哪里？什么类型的内容是首选？这些问题将帮助你总结出你可以用对话界面涵盖的领域。
*   **A/B 测试**例程将帮助找出最合适的演示。

## 机器学习模型训练

对话界面的另一大绊脚石是机器学习模型训练。虽然 ML 并不是每种类型的对话式 UI 都需要的——如果你的目标是提供个性化的体验和潜在客户——但设置正确的模式是必不可少的。

挑战是双重的。

*   首先，你需要教会机器人解释输入文本并给出相关的响应。
*   然后，您需要改进算法，帮助机器人适应特定的用户配置文件，以增加输出的个性化和相关性。

需要注意的是，这个挑战更多的是时间问题，而不是努力的问题。优化系统需要一些时间，但是一旦你通过了这个阶段——就万事大吉了。

要配置流畅的对话式用户界面，你需要结合描述性和预测性的机器学习算法。模型取决于用例。

## 自然语言处理配置

NLP 是对话界面的前端和中心。如果您的系统无法交付货物，这是一个坏消息，因为在内部深处缺少了一些东西，以至于用户得到了亵渎的“对不起，我不明白。”。

为了避免这种情况，您需要设置处理输入和交付输出的一致系统。

实际上，这意味着以下顺序:

*   **清除**输入的信息。这一过程包括:
*   删除标点符号
*   停用词移除
*   单词标记化
*   词干化、词汇化、向量化以**解释**信息
*   *决策制定组件*，“与外部服务集成以提交所请求的动作；
*   **响应的输出生成**。

简而言之，这就是最基本的对话式 UI 框架的工作方式。

## 隐私问题

随着对用户数据安全性的日益关注，维护个人数据的隐私和安全成为业务方面对话界面的重大挑战之一。

对话界面使用来自用户设备、电子邮件、联系人、搜索历史和其他数据的数据来提供适当的服务。

然而，鉴于所有这些操作经常通过第三方应用程序进行，隐私问题就悬而未决了。对话式 UI 总是有一个危险，那就是做一些不需要的额外工作，而且没有办法控制它。

如何解决这个问题？唯一可行的解决方案是添加以下解释:

*   你的对话界面助手是怎么操作的？
*   涉及到什么样的用户数据？

其他需要说明的重要细节包括:

*   用户数据是如何处理的？
*   是否化名？
*   信息在为其目的服务后是否会被处置，处置时间是什么时候？
